Authenticate client and client to server data flows

Documentation: https://bramp.github.io/js-sequence-diagrams

Title: Authenticate client data flow DDB CMS->Oauth2.login.bib.dk: Request authentication (Client_id + Client_secret). Oauth2.login.bib.dk->DDB CMS: Return authenticated (Access token) DDB CMS->Open Platform: Request authentication (Access token). Open Platform->Oauth2 authorization: Request authorization (Access token). Oauth2 authorization->Open Platform: Return authorized (Attributes) DDB CMS->Openlist: Request authentication (Access token). Openlist->Oauth2 authorization: Request authorization (Access token). Oauth2 authorization->Open Platform: Return authorized (Attributes)
